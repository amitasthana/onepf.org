AppDF - Application Description File

What is current project status?

  • Current version of AppDF specification is 1.0.
  • Two appstores Opera Mobile Store and Yandex.Store have AppDF publishing support in beta stage.
  • Several more appstores commited to support AppDF and are working on implementation now.
  • AppDF Editor is ready.
  • Command line tool for submitting AppDF to appstores is in beta.

Where can I submit AppDF files?

Currently you can submit them to Opera Mobile Store and Yandex.Store. Both have AppDF format support on their developer portals. Several more appstores are working on AppDF implementation.

How can I create AppDF file?

You can either follow AppDF specification or use AppDF Editor.

Can I do one appdf for all stores, including Google Play?

Yes, you can. When we created AppDF specification we carefully checked all the existing Android appstores to make sure AppDF format includes all needed information. AppDF also supports store specific sections so you can add custom information for any particular appstore.

What is difference between AppDF and mobile app agregators?

AppDF is a format that lets developers upload single distribution file to multiple appstores directly. In other words there is no middle man in AppDF approach, app developers publish their apps directly to the appstores not to some agregator.

Which Android appstores support AppDF format?

Currently Yandex.Store and Opera Mobile Store support AppDF.

How can I describe In-App Purchases in AppDF?

Right now AppDF does not support in-app purchases. It is not done yet because right now it is very difficult to create one APK file that will support in-app billing API of different appstores. We plan to support in-app purchases in the next version of AppDF which will be aligned with OpenIAB library that allows creating APK file that supports in-app billing of all the existing appstores.

OpenIAB - Open In-App Billing

What is current project status?

OpenIAB library is under development. We have finished a sample game that supports in-app billing of four different stores (Google Play, Amazon AppStore, Samsung Apps and SK T-Store) and now works on splitting in-app purchase code in a separate library. We will try to keep its interface as close to Google Play IAB API as possible. In the same time we have starting working on OpenIAB API that other stores could implement to support all the apps/games that use OpenIAB Lib. Join us to develop OpenIAB together.

Does OpenIAB support mobile operator billing?

It is a wrong question. All payments are processes directly by each appstore. There is no a middle man staying between the app developers and the appstores. OpenIAB will not do payments for the appstores. It is just an API how the apps communicate with the appstores to request in-app billing and get response.

What is difference between OpenIAB Lib and OpenIAB API?

OpenIAB Lib is a library that wraps in-app billing APIs of several appstores, you can integrate it once and your app will support in-app billing of all these stores. OpenIAB API is yet another in-app billing API that appstores can implement. OpenIAB API is integrated in OpenIAB Lib as one of the supported APIs. By implementing OpenIAB API the appstores will get automatic support of all the apps that use OpenIAB Lib.
