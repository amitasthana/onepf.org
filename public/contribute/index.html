How to Contribute

One Platform Foundation is an open source initiative to help Android developers to support all the appstores easily. There are a number of ways to get involved. Even if you have never contributed code to an open source project before, if you share our passion to make Android developer life easier to you welcome!

For Android App Developers
  1. First of all create AppDF files for your apps and publish them to the appstores that support AppDF
  2. If you have found a bug in AppDF Editor or found an idea how to improve the editor please report it
  3. If you know JavaScript and want to help improving AppDF editor please join AppDF Editor project
  4. If your game/app uses in-app purchases you may be interested joining OpenIAB project to make it easier to support IAB of different stores
For AppStores
  1. Start from subscribe to AppDF mailing list to know about the specification updated
  2. Read AppDF specification and support AppDF on your appstore, check AppDF parsers that could help you implementing AppDF support easier
  3. Check what is in roadmap of the next version of AppDF standard
  4. You can propose updates for the next version of AppDF
  5. If you have not implemented in-app purchases yet you can join OpenIAB project and help working on common OpenIAB API for appstores
Reporting a Bug

We use GitHub for bug tracking. You can use it to report bugs in AppDF Editor, AppDF specification, this website, the tools as well as publish your feature requests.

  1. Open AppDF issues list at GitHub
  2. Make sure you have a GitHub account, if not create one
  3. Check if bug you want to report is already reported
  4. If not create a new issue
  5. Please describe your environment (OS, browser, etc) and step-by-step instructions how to reproduce the bug, please also attached screenshots if possible
Joining AppDF Editor Project

AppDF Editor is a tool for creating AppDF files. It is a client-side HTML5 page with a lot of JavaScript. AppDF Editor is already working but there is always room for improvements.

  1. Make sure you have a GitHub account, if not create one
  2. Check the list of open tasks, check if any of these tasks is interesting for you, send a message to AppDF mailing list how you want to help.
  3. Fork the project at GitHub
  4. Please follow used code style when writing your code
  5. Make a pull request with your features
Joining OpenIAB Project

OpenIAB project is in early stage we are just creating a sample app that supports in-app billing of all the existing appstores and designing Open IAB API for the appstores.

  1. If you are an Android app developer check the list of open tasks, check if any of these tasks is interesting for you, send a message to OpenIAB mailing list how you want to help.
  2. If you are an appstore and already support In-App Billing then most probably we are already working on supporting your SDK in OpenIAB library, and your help is very welcome since you know your SDK better than anyone else! Just contact us by joining OpenIAB mailing list.
  3. If you are an appstore and do not support In-App Billing yet but plan to support it then we will be glad working with your on creating a common OpenIAB API and SDK. Joining OpenIAB mailing list to be involved in OpenIAB API development.
